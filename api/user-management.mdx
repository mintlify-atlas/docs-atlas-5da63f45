## Overview

User management covers the current user's profile, admin-facing user CRUD, and the role/group system used to control access throughout the API.

Every authenticated user has a `UserProfile` linked one-to-one to their Django `User` record. The profile holds extended fields (phone, address, bio, province, etc.) and is created automatically if it does not yet exist when first accessed.

### UserProfile model

| Field | Type | Description |
|---|---|---|
| `user` | OneToOne → User | The linked Django user |
| `phone` | string (max 20) | Contact phone number |
| `address` | string (max 255) | Street address |
| `birth_date` | date | Date of birth |
| `profile_picture` | image | Uploaded to `profile_pics/` |
| `bio` | text | Short biography |
| `roles` | ManyToMany → Group | Profile-level role assignments |
| `province` | ForeignKey → Provinces | Associated province (nullable) |
| `created_at` | datetime | Auto-set on creation |
| `updated_at` | datetime | Auto-updated on save |

## GET /api/auth/profile/

Returns the profile of the currently authenticated user. Creates the profile record automatically if it does not exist.

<Note>
  Requires authentication. Include your access token in the `Authorization: Bearer <token>` header.
</Note>

### Response

<ResponseField name="username" type="string">
  The user's username (read-only, sourced from the linked User).
</ResponseField>

<ResponseField name="email" type="string">
  The user's email address (read-only, sourced from the linked User).
</ResponseField>

<ResponseField name="phone" type="string">
  Contact phone number.
</ResponseField>

<ResponseField name="address" type="string">
  Street address.
</ResponseField>

<ResponseField name="birth_date" type="string">
  Date of birth in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="profile_picture" type="string">
  URL to the uploaded profile image.
</ResponseField>

<ResponseField name="bio" type="string">
  Short biography.
</ResponseField>

<ResponseField name="roles" type="object[]">
  Profile-level role objects.
  <Expandable title="role object">
    <ResponseField name="id" type="integer">Group ID.</ResponseField>
    <ResponseField name="name" type="string">Group name.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="user_roles" type="object[]">
  Roles assigned directly on the User object (Django groups).
  <Expandable title="user_role object">
    <ResponseField name="id" type="integer">Group ID.</ResponseField>
    <ResponseField name="name" type="string">Group name.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="province" type="integer">
  ID of the associated province.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 datetime when the profile was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 datetime of the last profile update.
</ResponseField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://your-api.example.com/api/auth/profile/ \
  --header 'Authorization: Bearer <your_access_token>'
```

```javascript JavaScript (fetch)
const response = await fetch('https://your-api.example.com/api/auth/profile/', {
  headers: { Authorization: `Bearer ${accessToken}` },
});

const profile = await response.json();
```

```javascript JavaScript (axios)
const { data } = await axios.get('/api/auth/profile/', {
  headers: { Authorization: `Bearer ${accessToken}` },
});
```

</CodeGroup>

**200 response**

```json

  "email": "juan@example.com",

  "address": "San José, Costa Rica",

  "bio": "Marine conservation enthusiast.",
  "roles": [{"id": 2, "name": "client"}],
  "user_roles": [{"id": 2, "name": "client"}],
```

## PUT /api/auth/profile/

Updates the current user's profile. Accepts `multipart/form-data` to support profile picture uploads. All fields are optional (partial update).

<Note>
  Requires authentication.
</Note>

### Request body

<ParamField body="phone" type="string">
  Contact phone number (max 20 characters).
</ParamField>

<ParamField body="address" type="string">
  Street address (max 255 characters).
</ParamField>

<ParamField body="birth_date" type="string">
  Date of birth in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="profile_picture" type="file">
  New profile image. Replaces any existing image. Stored under `profile_pics/`.
</ParamField>

<ParamField body="bio" type="string">
  Short biography.
</ParamField>

<ParamField body="province" type="integer | string">
  Province ID, or the province name as a string. If passed as a string, the API resolves it to the matching province record.
</ParamField>

### Response

Returns the full updated profile object. Same structure as `GET /api/auth/profile/`.

<CodeGroup>

```bash cURL
curl --request PUT \
  --url https://your-api.example.com/api/auth/profile/ \
  --header 'Authorization: Bearer <your_access_token>' \
  --form 'phone=+506 8888-5678' \
  --form 'bio=Updated bio text.' \
  --form 'province=2' \
  --form 'profile_picture=@/path/to/new_photo.jpg'
```

```javascript JavaScript (fetch)
const formData = new FormData();
formData.append('phone', '+506 8888-5678');
formData.append('bio', 'Updated bio text.');
formData.append('province', 2);
formData.append('profile_picture', fileInput.files[0]);

const response = await fetch('https://your-api.example.com/api/auth/profile/', {
  method: 'PUT',
  headers: { Authorization: `Bearer ${accessToken}` },
  body: formData,
});

const updatedProfile = await response.json();
```

</CodeGroup>

## GET /api/auth/user_profile/

Returns a list of all user profiles.

<Note>
  Requires authentication and the `IsAuthenticatedAndRole` permission. Access is restricted to authenticated users; set `required_role` on the view to limit by group.
</Note>

### Response

Returns an array of profile objects with the same structure as `GET /api/auth/profile/`.

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://your-api.example.com/api/auth/user_profile/ \
  --header 'Authorization: Bearer <your_access_token>'
```

```javascript JavaScript (fetch)
const response = await fetch('https://your-api.example.com/api/auth/user_profile/', {
  headers: { Authorization: `Bearer ${accessToken}` },
});

const profiles = await response.json();
```

</CodeGroup>

## GET /api/auth/user_profile/\{id\}/

Returns the profile for a specific user by their **user ID** (not profile ID).

<Note>
  Requires authentication.
</Note>

### Path parameters

<ParamField path="id" type="integer" required>
  The Django User primary key.
</ParamField>

### Response

Returns a single profile object. Same structure as `GET /api/auth/profile/`.

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://your-api.example.com/api/auth/user_profile/42/ \
  --header 'Authorization: Bearer <your_access_token>'
```

```javascript JavaScript (fetch)
const response = await fetch('https://your-api.example.com/api/auth/user_profile/42/', {
  headers: { Authorization: `Bearer ${accessToken}` },
});

const profile = await response.json();
```

</CodeGroup>

**404 response** — user not found

```json
{
  "error": "User not found"
}
```

## GET /api/auth/users/

Returns a paginated list of all Django users. This is an admin-facing endpoint.

<Note>
  Requires authentication via `IsAuthenticatedAndRole`. Only accessible to users with the appropriate role (configured via `required_role` on the view).
</Note>

### Response

Returns an array of user objects serialized by `RegisterSerializer`.

<ResponseField name="username" type="string" />
<ResponseField name="email" type="string" />
<ResponseField name="first_name" type="string" />
<ResponseField name="last_name" type="string" />
<ResponseField name="roles" type="integer[]">
  List of group IDs.
</ResponseField>
<ResponseField name="profile" type="object">
  Nested profile data if present.
</ResponseField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://your-api.example.com/api/auth/users/ \
  --header 'Authorization: Bearer <your_access_token>'
```

```javascript JavaScript (axios)
const { data } = await axios.get('/api/auth/users/', {
  headers: { Authorization: `Bearer ${accessToken}` },
});
```

</CodeGroup>

## DELETE /api/auth/users/\{id\}/delete/

Permanently deletes a user account.

<Warning>
  This action is irreversible. Deleting a user removes their account and all associated profile data via cascading database deletion.
</Warning>

<Note>
  Requires authentication and appropriate role access.
</Note>

### Path parameters

<ParamField path="id" type="integer" required>
  The user's primary key.
</ParamField>

<CodeGroup>

```bash cURL
curl --request DELETE \
  --url https://your-api.example.com/api/auth/users/42/delete/ \
  --header 'Authorization: Bearer <your_access_token>'
```

```javascript JavaScript (fetch)
const response = await fetch('https://your-api.example.com/api/auth/users/42/delete/', {
  method: 'DELETE',
  headers: { Authorization: `Bearer ${accessToken}` },
});
```

</CodeGroup>

## Roles and permissions

The platform uses Django's built-in Group model for role management. Every user can belong to one or more groups, which control access to role-restricted endpoints.

### GET /api/auth/roles/

Lists all available roles (Django groups).

<ResponseField name="id" type="integer">Group ID.</ResponseField>
<ResponseField name="name" type="string">Group name.</ResponseField>
<ResponseField name="permissions" type="integer[]">List of permission IDs assigned to the group.</ResponseField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://your-api.example.com/api/auth/roles/ \
  --header 'Authorization: Bearer <your_access_token>'
```

```javascript JavaScript (fetch)
const response = await fetch('https://your-api.example.com/api/auth/roles/', {
  headers: { Authorization: `Bearer ${accessToken}` },
});

const roles = await response.json();
```

</CodeGroup>

**200 response**

```json
[
  {"id": 1, "name": "admin", "permissions": [1, 2, 3]},
  {"id": 2, "name": "client", "permissions": []},
  {"id": 3, "name": "researcher", "permissions": [4, 5]}
]
```

### POST /api/auth/roles/create/

Creates a new role.

<Note>
  Requires authentication.
</Note>

<ParamField body="name" type="string" required>
  The name for the new role.
</ParamField>

### PUT /api/auth/permissions/\{id\}/update/

Replaces the permission set for a role by group ID.

<Note>
  Requires authentication and `IsAuthenticatedAndRole`.
</Note>

<ParamField path="id" type="integer" required>
  The group ID whose permissions you are updating.
</ParamField>

<ParamField body="permissions" type="integer[]" required>
  Full list of Django permission IDs to assign to this group. Replaces the existing set.
</ParamField>

<CodeGroup>

```bash cURL
curl --request PUT \
  --url https://your-api.example.com/api/auth/permissions/1/update/ \
  --header 'Authorization: Bearer <your_access_token>' \
  --header 'Content-Type: application/json' \
  --data '{"permissions": [1, 2, 3, 7]}'
```

```javascript JavaScript (fetch)
const response = await fetch('https://your-api.example.com/api/auth/permissions/1/update/', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    Authorization: `Bearer ${accessToken}`,
  },
  body: JSON.stringify({ permissions: [1, 2, 3, 7] }),
});
```

</CodeGroup>

## Password reset flow

The full password reset sequence involves two endpoints:

<Steps>
  <Step title="Request a reset token">
    Call `POST /api/auth/forgot-password/` with the user's email address. The response contains a `uid` and `token`.

    See [forgot password](/api/authentication#post-apiauthforgot-password) for full details.
  </Step>
  <Step title="Submit the new password">
    Call `POST /api/auth/reset-password-confirm/` with the `uid`, `token`, and the `new_password`.

    The token is single-use. On success the user's password is updated and the token is invalidated.

    See [reset password confirm](/api/authentication#post-apiauthreset-password-confirm) for full details.
  </Step>
</Steps>
